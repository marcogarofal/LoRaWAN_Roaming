How the Integrated Flow Works
Imagine a packet arrives from a LoRaWAN device that belongs to Tim's network (NetID 02), but is received by Vodafone's gateway (NetID 01).
In the traditional world, that packet would simply be discarded because it doesn't belong to Vodafone's network. But in the roaming world, Vodafone should be able to forward it to Tim in exchange for compensation.
With your system:

1)The device transmits the packet via radio
2)Vodafone's physical gateway receives it and sends it to the Gateway Bridge (via UDP port 1700 or directly via MQTT)
3)ChirpStack's standard Gateway Bridge converts it to JSON format and publishes it on MQTT
4)Your custom Gateway Bridge (with modified api.go) has subscribed to the same MQTT topics, so it receives a copy of the message
5)It extracts the DevAddr from the LoRaWAN frame (which is encrypted) and calculates the NetID by doing a bit shift: the NetID is the first 7 bits of the DevAddr
6)It sees that the NetID is "02", so it's not a packet from the local network (which would be "01")
7)At this point, instead of discarding, it makes an HTTP call to the Python blockchain service: "Hey, I have a packet with NetID 02, can I forward it?"
8)The Python service receives the request, searches the on-chain catalog for who is the provider with NetID 02 (finds Tim), and calls Vodafone's Smart Contract asking: "Is there an active SLA with Tim?"
9)Vodafone's Smart Contract on Algorand executes the logic on-chain: if it's the first time, it automatically initiates a handshake with Tim's Smart Contract. Tim's Smart Contract responds automatically (it's all on-chain code, no server to manage): it checks if the price Vodafone offers is acceptable, if the threshold is acceptable, and responds "Accept" or "Reject"
10)The response returns to the Python service, which passes it to the Go Gateway Bridge
11)The Gateway Bridge sees "Accept", so it forwards the packet on MQTT to ChirpStack, which will process it as if it had arrived normally
12)The Python service also calls the function to increment the counter on blockchain: "I forwarded 1 packet for Tim"
13)This is recorded immutably on the blockchain
14)When the counter reaches the threshold (e.g., 5 packets), the Smart Contract automatically executes a payment from Tim to Vodafone. If the payment succeeds, the counter resets and the SLA continues. If it fails, the SLA goes into "Banned" state and future packets are blocked.
